Marketplace Rules

Перед публикацией ваших скриптов и конфигов в Marketplace, пожалуйста, внимательно ознакомьтесь с правилами. Их нарушение приведет к отклонению заявки.

  • Отсутствие вредоносного функционала: Запрещены любые вебхуки (Discord, Telegram и др.), логгеры, IP-грабберы и скрипты, угрожающие безопасности пользователей либо ломающие Vantality.
  • Названия и описание: Название скрипта/конфига должно соответствовать его реальному функционалу. В названии и описание не должно быть оскорбительного контекста..
  • Уважение авторского права: Выдавать чужие скрипты за свои запрещено. Если вы модифицировали чужой открытый код, укажите оригинального автора в описании
  • Стабильность: Перед отправкой убедитесь, что ваш скрипт не крашит клиент игры и корректно выгружается за собой при удалении.

Первые шаги к разработке

В этом блоке расписано, как начать разрабатывать собственные скрипты для Luau API и "куда тыкать":

  • Где находятся скрипты и конфиги?: Чтобы найти папку со скриптами, перейдите по следующему пути внутри папки вашего эксплойта: /workspace/Vantality/scripts. Папка с конфигами находится по пути: /workspace/Vantality/Config/Main.
  • Как начать создавать свои собственные скрипты?: Перед созданием первого скрипта ознакомьтесь с документацией ниже.
    После изучения перейдите по пути /workspace/Vantality/scripts и создайте файл с расширением .luau
  • Как добавить свой скрипт в маркетплейс?: После того как вы создадите свой скрипт, скопируйте его код и опубликуйте на GitHub, Pastebin или любом другом сервисе с поддержкой raw-ссылок. После этого подайте заявку на публикацию в Telegram-боте:
    ahkgbs_bot. ( /start -> Мой профиль -> Для разработчиков -> Добавить script )
  • Как добавить свой конфиг в маркетплейс?: Сохраните настройки вашего конфигов, затем перейдите по пути /workspace/Vantality/Config/Main и откройте файл вашего конфига в Блокноте. Скопируйте его содержимое и опубликуйте на GitHub, Pastebin или любом другом сервисе с поддержкой raw-ссылок. После этого подайте заявку на публикацию в Telegram-боте:
    ahkgbs_bot ( /start → Мой профиль → Для разработчиков → Добавить config )
  • Если у вас возникнет больше вопросов, вы можете обратиться в чат поддержки нашего телеграм бота: ahkgbs_bot
Vantality.CreateSection (title: string) returns: SectionProxy

Инициализирует новую секцию. API само подбирает колонку для расположения секции.

local UI = Vantality.CreateSection("General Settings")

Создание элементов (UI Methods)

Важно: Все созданные вами секции и элементы будут автоматически распределены по колонкам внутри вкладки LUAU.

AddParagraph

Обычный текстовый блок для вывода информации. Автоматически подстраивается под высоту текста.

UI:AddParagraph({ 
    Title = "Information", 
    Content = "Скрипт разрабатывается.." 
})

AddInput

Текстовое поле ввода.
Callback: срабатывает при потере фокуса (нажатии Enter/клике вне поля).

UI:AddInput({ 
    Name = "Target Player", 
    PlaceholderText = "Введите ник...", 
    Default = "", 
    Callback = function(text) end 
})

AddToggle

Переключатель (Чекбокс).
Risky: оранжевый текст, выделяет важный элемент.
Option: включает вложенное меню (шестеренку).

UI:AddToggle({ 
    Name = "God Mode", 
    Default = false, 
    Risky = true,
    Option = true,
    Callback = function(state) end 
})

AddSlider (Integer)

Слайдер для целых чисел. Используйте Step для шага при настройке и Type для суффикса (%, s, studs).

UI:AddSlider({ 
    Name = "WalkSpeed", 
    Range = {16, 200}, 
    Default = 16, 
    Step = 1,
    Type = " studs",
    Callback = function(v) end 
})

AddSlider (Precise)

Для дробных значений (Float).
Precise: кол-во знаков после запятой.

UI:AddSlider({ 
    Name = "Jump Height", 
    Range = {0, 1}, 
    Default = 0.5, 
    Precise = 2,
    Callback = function(v) end 
})

AddDropdown

Одиночный выбор строки из списка Values. Возвращает строку.

UI:AddDropdown({ 
    Name = "Target Bone", 
    Values = {"Head", "Torso", "Random"}, 
    Default = "Head",
    Callback = function(v) end 
})

AddDropdown (Multi)

Множественный выбор.
Callback: возвращает таблицу { = bool }.

UI:AddDropdown({ 
    Name = "Filters", 
    Values = {"Players", "NPCs"}, 
    Default = { = true, = false},
    Multi = true,
    Callback = function(tbl) end 
})

AddColorPicker

Выбор цвета.
Transparency: поддержка альфа-канала (0-1).

UI:AddColorPicker({ 
    Name = "ESP Color", 
    Default = Color3.fromRGB(255, 0, 0),
    Transparency = 0.5,
    Callback = function(color, alpha) end 
})

AddKeybind

Привязка клавиши. Поддерживает Enum и бинд пользователя.

UI:AddKeybind({ 
    Name = "Flight Key", 
    Default = Enum.KeyCode.F,
    Callback = function(key) end 
})

AddButton

Простое действие при нажатии.

UI:AddButton({ 
    Name = "Reset All", 
    Callback = function() end 
})

Методы элементов (Response Object)

Каждый метод создания элемента (например, AddToggle или AddSlider) возвращает объект, с помощью которого можно управлять элементом из кода:

  • :GetValue() — возвращает текущее значение элемента. Позволяет не создавать лишних глобальных переменных для хранения стейтов.
  • :SetValue(value, force) — устанавливает новое значение. Если параметр force = true, принудительно вызовет функцию Callback.
  • :Rename(string) — изменяет отображаемое имя элемента в интерфейсе.
Прокси объекты и Nested Option

Если элемент создан с параметром Option = true, возвращаемый объект содержит свойство .Option. Это полнофункциональный контейнер Elements, унаследовавший методы секции. Из него можно вызывать Add... методы напрямую.

local master = UI:AddToggle({ Name = "Main Switch", Option = true })

-- Добавляем элементы внутрь шестеренки:
if master.Option then
    master.Option:AddButton({ Name = "Internal Action", Callback = function() end })
end

Служебные функции и События

Vantality.Bind (signal: RBXScriptSignal, callback: function) returns: RBXScriptConnection

Важно: Никогда не делайте .Connect напрямую к сигналам игры. Используйте Vantality.Bind — это гарантирует, что соединение будет разорвано автоматически, когда скрипт выгрузится.
Метод возвращает стандартный объект соединения, который можно отключить вручную до выгрузки.

local myLoop = Vantality.Bind(game:GetService("RunService").RenderStepped, function(dt)
    -- Ваш цикл
end)

-- Отключение вручную, если это необходимо
-- myLoop:Disconnect()
Vantality.Notify (config: table)

Отправка всплывающего уведомления в интерфейсе Vantality.

Vantality.Notify({ 
    Title = "Success", 
    Content = "Settings Applied!", 
    Icon = "check-circle", 
    Duration = 5 
})

Жизненный цикл и OnUnload

Когда пользователь нажимает "Unload" или удаляет ваш скрипт, API вызывает глобальную функцию OnUnload, если она определена. С её помощью вы должны вернуть игру в исходное состояние (например, удалить парты или восстановить параметры).

cleanup_logic.lua
-- Сохраняем оригинальные параметры
local oldFog = game.Lighting.FogEnd

function OnUnload()
    -- Обязательно возвращаем всё как было
    game.Lighting.FogEnd = oldFog
    print("Script Resources Cleared")
end

ГОТОВЫЙ СКРИПТ

Пример всех основных возможностей API в одном файле.

example_api.lua 
-- 1. создаём секцию
-- CreateSection автоматически подберет колонку во вкладке LUAU.
local UI = Vantality.CreateSection("Luau API example")

-- 2. параграфы (текст для инфы внутри секции)
UI:AddParagraph({
	Title = "Welcome to Vantality Luau API", -- жирный заголовок параграфа
	Content = "показывает весь функционал API. юзайте GetValue() и SetValue(), чтобы не плодить миллион локальных переменных." -- основной текст в параграфе (описание)
})

-- 3. текстовое поле (input)
local SpooferInput = UI:AddInput({
	Name = "Name Spoofer", -- название инпута
	PlaceholderText = "введите ник...", -- серый текст, который виден, пока поле ввода пустое
	Default = "grab1s", -- дефолтный текст при загрузке скрипта
	Callback = function(text) -- Callback вызывается, когда юзер нажимает Enter или кликает вне поля ввода
		print(">> новый ник для спуфера:", text)
	end
})

-- 4. переключатель (Toggle) и управление (get/set)
local OverlayToggle = UI:AddToggle({
	Name = "Screen Overlay", -- название переключателя
	Default = false, -- изначальное состояние (тут выключено)
	Option = true, -- включает доп. меню справа (появляется иконка шестеренки)
	Risky = true, -- красит текст в оранжевый (типо важная или опасная функция, юзайте как вам угодно)
	Callback = function(state) -- возвращает true/false при клике
		print("Overlay state:", state)
	end
})

-- закидываем доп. настройки ВНУТРЬ тоггла (в шестеренку).
-- ВАЖНО: всегда чекайте наличие .Option перед добавлением
if OverlayToggle.Option then
	OverlayToggle.Option:AddButton({
		Name = "Force Disable",
		Risky = true, -- кнопки внутри шестеренки тоже поддерживают Risky
		Callback = function()
			-- меняем значение ползунка/тоггла напрямую через метод.
			-- второй аргумент (true) принудительно триггерит Callback самого тоггла
			OverlayToggle:SetValue(false, true)
			-- переименовываем элемент в реальном времени
			OverlayToggle:Rename("Screen Overlay (Off!)")
		end
	})
end

-- 5. слайдеры (int и float)
local FovSlider = UI:AddSlider({
	Name = "Camera FOV",
	Range = {70, 120}, -- задаём лимит: {минимум, максимум}
	Default = 90, -- дефолтное значение ползунка
	Option = false, -- может быть использовано тут
	Risky = false, -- может быть использовано тут
	Step = 1, -- шаг (int). если 10, то слайдер будет меняться так: 0, 10, 20, 30...
	Type = "°", -- суффикс, добавляется к цифре визуально (например: 90°)
	Callback = function(val) -- возвращает число
	end
})

local StretchSlider = UI:AddSlider({
	Name = "Screen Stretch",
	Range = {0, 1},
	Default = 0.5,
	Option = false, 
	Risky = false, 
	Precise = 2, -- сколько знаков после запятой (0.50). (используется вместо Step для float-значений)
	Callback = function(val)
	end
})

-- 6. выпадающий список (dropdown)
-- обычный выбор одного пункта
local SkyboxDropdown = UI:AddDropdown({
	Name = "Custom Sky",
	Values = {"Night", "Vaporwave", "Blood Moon"}, -- Список доступных вариантов (массив строк)
	Default = "Night", -- вариант, выбранный по умолчанию (должен быть из списка Values)
	Option = false, -- может быть использовано тут
	Multi = false, -- если false - можно выбрать только один пункт
	Callback = function(val) -- возвращает строку ("Night")
	end
})

-- мульти-выбор
UI:AddDropdown({
	Name = "Render Filters",
	Values = {"Players", "Corpses", "Map Details"}, 
	Default = { ["Players"] = true, ["Corpses"] = false }, -- для Multi: указываем таблицу состояния пунктов
	Option = false, -- может быть использовано тут
	Multi = true, -- если true = можно включить несколько пунктов одновременно
	Callback = function(tbl) -- возвращает таблицу состояний: { Players = true, Corpses = false ... }
	end
})

-- 7. цвета и бинды клавиш
local ArrowColor = UI:AddColorPicker({
	Name = "Player Arrows Color",
	Default = Color3.fromRGB(255, 106, 133), -- дефолтный цвет
	Option = false, -- может быть использовано тут
	Transparency = 0.5, -- дефолтная прозрачность (альфа-канал от 0 до 1)
	Callback = function(color, alpha) -- возвращает (Color3, number)
	end
})

local CameraStrafeKey = UI:AddKeybind({
	Name = "Camera Strafe Key",
	Default = Enum.KeyCode.C, -- кнопка по умолчанию (можно юзать строки, например "C")
	Option = false, -- может быть использовано тут
	Callback = function(key) -- возвращает название нажатой клавиши
	end
})

-- 8. кнопки и система уведомлений
UI:AddButton({
	Name = "Apply Visuals", -- текст на кнопке
	Risky = false, -- может быть использовано тут
	Callback = function() -- срабатывает при каждом клике по кнопку
		Vantality.Notify({
			Title = "Visuals Loaded", -- заголовок уведомления
			Content = "визуалы применились", -- текст уведомления
			Icon = "check-circle", -- название иконки из библиотеки (см. раздел Icons)
			Duration = 3 -- сколько секунд висит на экране
		})
	end
})

-- 9. ЛОГИКА (ГЛАВНЫЙ ЦИКЛ)
-- юзаем :GetValue(), чтобы не срать глобальными переменными в коде скрипта.

Vantality.Bind(game:GetService("RunService").RenderStepped, function(dt)
	-- читаем актуальное состояние элементов напрямую
	if OverlayToggle:GetValue() then
		local sky = SkyboxDropdown:GetValue()
		local fov = FovSlider:GetValue()

		-- колорпикер возвращает таблицу (цвет + прозрачность)
		local arrowData = ArrowColor:GetValue()
		local rgb = arrowData.Color
		local alpha = arrowData.Transparency

		-- тут ваша математика под плейс (например, апдейт стрелочек на врагов)
	end
end)

-- 10. ВЫГРУЗКА (CLEANUP)
-- API само отрубит коннекты (Bind) и удалит секцию.
-- НО если вы спавните партиклы, локалпарты (Instance.new) или юзаете Drawing API:
-- ВЫ ОБЯЗАНЫ УДАЛИТЬ ИХ ТУТ САМИ.

-- пример кастомного объекта:
local IndicatorText = Drawing.new("Text")

function OnUnload()
	print(">> script unloaded via API.")

	-- чистим за собой мусор
	if IndicatorText then
		IndicatorText:Remove()
	end
	-- возвращаем стейты игры в исходную
	-- game.Lighting.FogEnd = 100000
end
